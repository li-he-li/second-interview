# 测试报告

## 环境

- OS：Windows 11；Python 3.12.4；虚拟环境 `.venv`
- 依赖：pytest 9.1.1、pydantic 2.13.4、tiktoken 0.13.0、openai 2.43.0
- 配置目录：`config/`（默认）

## 稳定测试命令

```bash
.venv/Scripts/python.exe -m pytest tests/
```

- 临时目录由 `conftest.py` 的 `pytest_configure` 动态生成为项目内唯一子目录
  `.temp/pytest-<时间戳>-<pid>`，解决两个问题：
  1. 系统临时目录权限不足导致 `tmp_path` 写入失败；
  2. 固定单一 basetemp 复跑时清理旧 session 目录触发文件锁/权限错误。
- 命令行显式 `--basetemp=xxx` 仍被尊重；`.temp/` 已在 `.gitignore`。
- 已验证：连跑两次均 55 passed，无清理冲突。
